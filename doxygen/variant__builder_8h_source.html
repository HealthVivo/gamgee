192  /******************************************************************************
193  *
194  * Functions for setting individual/FORMAT fields in bulk
195  * (ie., setting all values for all samples at once)
196  *
197  * -These are much more efficient than the per-sample setters.
198  *
199  * -Setting by field index is more efficient than setting by field name,
200  * provided that you look up the indices ONCE in the header and cache
201  * them instead of looking them up for every record.
202  *
203  * -The bulk setting functions that take a vector by move/r-value reference
204  * are much more efficient than the bulk setting functions that take a vector
205  * by l-value reference, since the latter make a copy of the data.
206  *
207  * -The bulk setting functions that take a flattened (one-dimensional) vector
208  * are much more efficient than the functions that take a two-dimensional vector,
209  * but the flattened approach requires the user to pad sample values with
210  * missing/end of vector values manually, whereas with the two-dimensional
211  * approach the user does not need to worry about padding sample values.
212  *
213  * For example, with 4 samples and an integer individual field with a
214  * varied number of values per sample, you could pass in the following
215  * nested vector:
216  *
217  * { {1, 2}, {3}, {}, {5, 6, 7} }
218  *
219  * with each inner vector representing the values for one sample.
220  *
221  * However, to use the flattened vector API functions you'd have to pass in
222  * this equivalent single-dimensional vector:
223  *
224  * { 1, 2, int32_vector_end,
225  * 3, int32_vector_end, int32_vector_end,
226  * int32_missing_value, int32_vector_end, int32_vector_end,
227  * 5, 6, 7 }
228  *
229  * The advantage of the flattened approach is much greater efficiency
230  * in terms of data locality and memory allocations.
231  *
232  * -The Genotype setter functions require you to encode your genotype data
233  * with one of the Genotype::encode_genotype() functions before passing it in.
234  *
235  * For example, if you want to encode the genotype 0/1, create a vector with
236  * {0, 1}, pass it to Genotype::encode_genotype(), then pass it to the appropriate
237  * set_genotypes() function below.
238  *
239  ******************************************************************************/
