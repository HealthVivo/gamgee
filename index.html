What is Gamgee

Gamgee is a free open source C++14 library for the main file formats in genomics. Currently it supports reading and writing of SAM, BAM, CRAM, VCF, BCF, FASTA, FASTQ files. Gamgee is designed for high performance traversals of these files. It provides a selection of iterators and utility functions to load the entire file to memory. Gamgee uses semantic versioning 2.0 The API is free to change until version 1.x is released.

Dependencies

To build gamgee you will need a C++14 capable compiler. Currently Gamgee is tested on GCC-4.9 or Clang 3.4.

Gamgee dependes on the Boost Libraries, including Boost.Build V2 installed in your system. If your operating system has a package manager, simply install the full boost library or download them separately and install from source.

Building Gamgee

To build gamgee simply use b2 in the clone directory

$ git clone https://github.com/mauriciocarneiro/gamgee.git
$ cd gamgee
$ b2

To run tests in Gamgee simply run from the git clone root:

$ b2 test

You can specify a specific compiler with the toolset argument to b2.

$ b2 toolset=clang

You can force-build all sources even if there are no changes (this is the only way to force a rebuild of the submodules) with: 

$ b2 -a

You can use multiple threads to build faster with -j N where N is the number of threads you want to use. Here is a typical command: 

$ b2 -aj4 test

Contributing to Gamgee

To contribute to Gamgee, simply fork this repository, make your suggested changes and open a pull request. Your pull request will be evaluated by the maintainers of the library and the code-review will happen in the pull-request interface of Github.

When you fork this repository you are also automatically set up to have continuous integration builds happening at travis-ci. You can use the travis badge in the README.md file in your forked project's github page to go directly to travis-ci and see the build of your code. Only builds with a passing status will be considered for code review.

Every functionality added to Gamgee should have an accompanying unit test added in the test directory.
